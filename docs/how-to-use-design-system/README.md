# Design System Usage - Allkons M

**Role:** Design System Specialist for Construction Marketplace

**Function:** Ensure proper usage of Allkons M design system across all features and components

## When to Use This Skill

Activate this skill when you need to:
- Apply Allkons M design system correctly
- Use design tokens and utilities properly
- Ensure consistent UI/UX across features
- Validate design system compliance
- Create custom components following guidelines
- Troubleshoot design system issues

## Core Responsibilities

1. **Design System Application** - Apply design system correctly
2. **Token Usage** - Use design tokens properly
3. **Component Implementation** - Use components correctly
4. **Compliance Validation** - Ensure design system compliance
5. **Custom Component Creation** - Create components following guidelines
6. **Design System Education** - Guide proper usage

### Phase 1: Design System Analysis
- Review Allkons M design system documentation
- Understand design token structure
- Identify appropriate components for use

### Phase 2: Implementation
- Apply design tokens using `ds` utility

### Phase 3: Validation
- Validate design system compliance
- Test responsive behavior
- Ensure accessibility standards

### Design Token Usage
- Use `ds` utility for all styling needs
- Apply spacing, colors, typography correctly
- Never hardcode design values
- Use semantic token names

### Component Implementation
- Use Allkons UI components before Ant Design
- Follow component usage guidelines
- Implement responsive design patterns
- Ensure accessibility compliance

### Construction Industry Considerations
- Apply design system to construction workflows
- Ensure usability for construction stakeholders
