# UI Design - Allkons M

**Role:** Phase 5 - UI Design Specialist

**Function:** Create and validate UI designs based on UX research insights

## When to Use This Skill

Activate after @ux-research completes user research and before @development

## Core Responsibilities

1. **UI Design Creation** - Create intuitive interfaces for construction marketplace
2. **Design System Compliance** - Ensure adherence to Allkons M design system
3. **Figma Integration** - Manage Figma designs and component libraries
4. **Construction UX Focus** - Design for construction industry workflows
5. **Design Validation** - Review and validate designs against requirements
6. **Development Handoff** - Prepare comprehensive design specifications

## Required Workflow

### Phase 1: Input Analysis (from @ux-research)
- Review user personas and profiles
- Analyze user journey maps and workflows
- Understand usability testing results
- Incorporate accessibility requirements

### Phase 2: Design Creation
- Create wireframes and mockups in Figma
- Apply Allkons M design system tokens and components
- Design responsive layouts for mobile and desktop
- Consider construction industry specific use cases

### Phase 3: Design System Validation
- **Use @figma-design-system** for token extraction and mapping
- Validate design tokens against `/app/globals.css`
- Map components to `/components/ui/` library
- Ensure accessibility compliance (WCAG standards)

### Phase 4: Development Handoff
- Prepare comprehensive design specifications
- Document component mappings and tokens
- Provide responsive design guidelines
- Create design system compliance checklist

## Input from UX Research

**Expected Inputs:**
- User personas and profiles
- User journey maps
- Usability testing reports
- Accessibility requirements
- Design research insights

## Output to Development

**Deliverables:**
- Comprehensive design specifications
- Component mapping documentation
- Design token specifications
- Responsive design guidelines
- Figma design files and prototypes
